# n8n - TestBot Deployment

TestBot-compatible deployment scripts for n8n workflow automation platform.

## Files

- **setup.sh** - Starts n8n, creates the owner account on first boot, and mints an API key into `.api-key`
- **get-token.sh** - Prints the API key from `.api-key` (for use as `X-N8N-API-KEY` header)
- **docker-compose.yml** - Service configuration
- **.api-key** - Generated by setup.sh, gitignored

## TestBot Configuration

```yaml
- uses: skyramp/testbot@v1
  with:
    skyramp_license_file: ${{ secrets.SKYRAMP_LICENSE }}
    cursor_api_key: ${{ secrets.CURSOR_API_KEY }}
    target_setup_command: './deployment-testbot/setup.sh'
    target_ready_check_command: 'curl -f http://localhost:5678/healthz'
    auth_token_command: './deployment-testbot/get-token.sh'
    target_teardown_command: 'docker-compose -f deployment-testbot/docker-compose.yml down'
```

## Application Details

- **Port:** 5678
- **API Base:** http://localhost:5678/api/v1
- **Web UI:** http://localhost:5678
- **Health Endpoint:** /healthz
- **Auth Type:** API key (`X-N8N-API-KEY` header)
- **Owner account:** `testbot@example.com` / `TestBotPass123!` (override via `N8N_OWNER_EMAIL` / `N8N_OWNER_PASSWORD` env vars before running setup.sh)

## API Endpoints

- `GET /api/v1/workflows` - List workflows
- `POST /api/v1/workflows` - Create workflow
- `GET /api/v1/workflows/:id` - Get workflow
- `PUT /api/v1/workflows/:id` - Update workflow
- `DELETE /api/v1/workflows/:id` - Delete workflow
- `POST /api/v1/workflows/:id/activate` - Activate workflow
- `GET /api/v1/executions` - List executions

## Manual Testing

```bash
# Start services + create owner + mint API key
./setup.sh

# Read the API key
TOKEN=$(./get-token.sh)

# Test API
curl -H "X-N8N-API-KEY: $TOKEN" http://localhost:5678/api/v1/workflows

# Stop services (keep volume / API key for next run)
docker compose down

# Full teardown (also drops the persisted owner account and API key)
docker compose down -v && rm -f .api-key
```
